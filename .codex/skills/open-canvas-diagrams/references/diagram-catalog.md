# Diagram Catalog

Use these defaults unless the user asks for a specific diagram.

## System architecture

Use for services, tools, databases, integrations, and data flow between components.

Good nodes:
- `service`
- `database`
- `container`
- `actor`
- `document`

## Flowchart

Use for step-by-step logic, workflows, funnels, or procedures.

Good nodes:
- `process`
- `decision`
- `document`
- `event`

## Mind map

Use for brainstorming, concept clustering, research synthesis, or idea exploration.

Good nodes:
- `container`
- `process`
- `note`
- `event`

## Sequence

Use for back-and-forth interactions over time, especially with lanes or actors.

Good nodes:
- `actor`
- `service`
- `process`
- `event`

## Org chart

Use for teams, ownership, reporting, or responsibility trees.

Good nodes:
- `actor`
- `container`
- `service`

## Timeline

Use for milestones, launches, campaigns, phased work, or event sequences.

Good nodes:
- `event`
- `process`
- `document`
